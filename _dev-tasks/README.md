# _dev-tasks

This folder holds advisory maintenance tasks generated by `_dev-system`.

Human-authored product, workflow, and cleanup tasks live in `_tasks/`. Read `_tasks/TASKS.md` before creating or completing those tasks.

Run:

```bash
./_scripts/run_dev_analyzer.sh
```

Then pick one task file and handle it with a focused change. After the change, run:

```bash
./ttt.sh test
./_scripts/run_dev_analyzer.sh
```

Tasks are intentionally advisory. Some large modules are valid coordination points, but the generated notes are useful reminders about where future refactors should be careful.
